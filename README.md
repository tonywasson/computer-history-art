# Computer History Gitfiti: 2012-2025

This dedicated repository tells a fourteen-year story through dated,
intentional contribution art. The visual terms are Morpho, MOL, and PyMOL
inspired companion vocabulary. They are not ordinary engineering history,
scientific evidence, or action authority.

- 2012: GIT — Git becomes the durable ledger for code and history.
- 2013: NEURAL NET — Neural nets move from curiosity into a named working lane.
- 2014: STATS SAS — Statistics and SAS become a practical analysis chapter.
- 2015: SCD DDL — Slowly changing dimensions and DDL become structural craft.
- 2016: ELT ELT — ELT becomes the repeated operating pattern rather than an exception.
- 2017: OPTIMIZE — Optimization becomes a named habit: tune the route, not just the result.
- 2018: POOLING — Pooling becomes visible as a performance and coordination surface.
- 2019: CACHING — Caching becomes an explicit lever for latency and workload shape.
- 2020: TST — TST: transition-state vocabulary enters the visual bridge.
- 2021: FORZA — FORZA: governed recommendation defense keeps the crux visible.
- 2022: AWS — AWS becomes the named cloud chapter in the annual arc.
- 2023: ANSIBLE — Ansible becomes the automation lane for repeatable operations.
- 2024: GCP — GCP becomes the next cloud chapter in the operating map.
- 2025: EYRING — Eyring becomes the bridge term for the transition-state visual stack.
